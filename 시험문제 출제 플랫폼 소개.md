# 시험문제 출제 플랫폼 소개

> 간호학과 교과목 시험문제를 AI로 출제·편집·인쇄하는 단일 파일 웹 플랫폼
> **청암대학교 간호학과 · 정종필 교수 · imjp5678@scjc.ac.kr**
> 대상 교과목: **디지털리터러시와 AI활용**(1학년) · **AI융합 간호정보학**(3학년) — 학년·교과목명 직접 입력도 가능
> 파일: `index.html` · https://jp5678.github.io/exam-authoring/ | 작성일: 2026. 7. 2.

---

## 1. 개요

이 플랫폼은 학습내용(강의노트, 교재 목차, 수업자료 파일)을 입력하면 Claude AI가 시험 범위 안에서만 문항을 출제해 주는 도구입니다. 출제된 문제는 실제 대학 시험지 양식으로 렌더링되며, 화면에서 바로 수정한 뒤 인쇄(PDF)하거나 Word 파일로 저장할 수 있습니다. 별도의 설치나 서버 없이 **HTML 파일 하나를 더블클릭하는 것만으로 동작**합니다.

## 2. 주요 기능

### 출제 설정
| 기능 | 내용 |
|---|---|
| 학년도·학기 선택 | 항상 전년도부터 3년 후 학년도까지 표시, 1·2학기 |
| 학년 · 교과목 | 초기 미선택 상태에서 학년 드롭다운(1~4학년) 선택 + 교과목명 직접 입력, 기본 과목 원클릭 칩(1학년 디지털리터러시와 AI활용 · 3학년 AI융합 간호정보학) — 과목 정보가 시험지와 출제 프롬프트에 자동 반영 |
| 시험 구분 · 시험 일시 | 진단평가 / 형성평가 / 중간고사 / 기말고사(기본: 진단평가) + 시험 일시(날짜 캘린더 + 시각 드롭다운, 분은 10분 단위) — 구분은 시험지 표제에, 일시는 머리표 '일시' 칸에 자동 표기 |
| 유형별 문항 수·배점 | 5지선다 객관식, 참/거짓(진위형 O·X), 단답형, 주관식(서술형) 각각 문항 수와 문항당 배점 지정(초기값 0), 총점 자동 계산 |
| 난이도 분포 | 혼합(하30·중40·상30), 표준(하25·중50·상25), 하 위주, 중 위주, 상 위주 |
| 학습내용 입력 | 강의 주제·강의노트 텍스트 붙여넣기 |
| 수업자료 파일 첨부 | docx, pdf, pptx, xlsx/xls, csv, txt/md, hwpx — 브라우저 안에서 텍스트를 추출해 출제 범위에 포함 |

### 출제 방식 (2가지)
1. **Claude API 자동 출제** — Anthropic API 키를 입력하면 버튼 한 번으로 완성. 진행바로 출제 진행 상황 표시. 키는 브라우저에만 저장되며, **「키 삭제」 버튼**으로 언제든 지울 수 있어 공용 PC에서도 안전하게 사용.
2. **프롬프트 복사 출제** — API 키가 없어도 사용 가능. 완성된 출제 프롬프트를 복사해 claude.ai 등 AI 챗봇에 붙여넣고, AI의 답변(JSON)을 다시 붙여넣으면 시험지가 자동 생성.

### 시험지·정답지
- **실제 시험지 양식**: 학년도·시험명·교과목 표제, 학과/학년/일시(캘린더 선택 시 자동 표기)/반/학번/이름/배점/점수 기재란, 응시 유의사항, 유형별 섹션(Ⅰ객관식·Ⅱ참/거짓·Ⅲ단답형·Ⅳ서술형 — 사용한 유형만 자동 번호 부여), 단답 답안란·참/거짓 기입란·서술형 답안 괘선 포함
- **정답 및 해설지 자동 생성**: 객관식·참/거짓 정답표(문항 줄·정답 줄 교차 배열, O·X 표기), 단답형 정답과 복수 인정답안, 서술형 모범답안 + 항목별 채점기준(배점 포함)
- 각 문항에 **난이도(상·중·하)와 배점 배지** 표시 — 인쇄 시 난이도 배지는 자동 제외

### 편집·활용 도구
- **문항 클릭 즉시 수정**: 시험지 위에서 문항·보기·정답·해설을 바로 편집, 수정 내용은 인쇄·저장에 그대로 반영
- **문항·보기 섞기**: A/B형 시험지 제작용. 객관식 보기를 섞어도 정답표가 자동으로 따라감
- **인쇄 / PDF 저장**: 시험지와 정답지를 분리 인쇄, 문항이 페이지 중간에 잘리지 않도록 처리
- **Word(.doc) 저장**: 시험지 + 정답지가 한 파일로 다운로드
- **문제은행**: 출제한 시험을 브라우저에 저장, 목록에서 불러오기·삭제

## 3. 특징과 장점

### ① 설치가 필요 없는 단일 파일
HTML 파일 하나가 전부입니다. 서버, 데이터베이스, 프로그램 설치, 회원가입이 전혀 필요 없고 USB나 이메일로 파일을 옮겨도 그대로 동작합니다.

### ② 출제 범위 이탈 방지
출제 프롬프트에 "입력한 학습내용 안에서만 출제"를 강제하고, 교과목별 성격(1학년 교양 수준 / 3학년 전공 수준)과 어휘 수준을 지정합니다. 학습내용에 없는 내용은 출제되지 않도록 설계되었습니다.

### ③ 데이터가 외부로 나가지 않음
- 수업자료 파일의 텍스트 추출은 **전부 사용자 브라우저 안에서** 처리됩니다 (서버 업로드 없음)
- API 키와 문제은행은 사용자 컴퓨터의 브라우저 저장소(localStorage)에만 보관되며, API 키는 「키 삭제」 버튼으로 즉시 제거할 수 있습니다
- 외부 통신은 출제 시 Anthropic API 호출(API 모드 선택 시)과 폰트·추출 라이브러리 로드뿐입니다

### ④ 교수자 검토 중심 워크플로
AI가 초안을 만들고 최종 판단은 교수자가 합니다. 화면에서 바로 고치고, 섞고, 저장하는 흐름이라 별도 편집 프로그램을 오갈 필요가 없습니다.

### ⑤ API 키가 없어도 사용 가능
"프롬프트 복사 출제" 모드로 claude.ai(무료 계정 포함)나 다른 AI 챗봇을 활용할 수 있어 비용·계정 부담 없이 시작할 수 있습니다.

### ⑥ 시험 실무에 맞춘 세부 기능
총점 자동 검산, 서술형 채점기준(루브릭) 자동 생성, 문항·보기 셔플과 정답표 자동 갱신, 객관식 부정형 문항 비율 제한(20% 이내) 등 실제 출제 규정을 반영했습니다.

## 4. 기술 스택

| 구분 | 기술 | 용도 |
|---|---|---|
| 구조 | HTML5 단일 파일 (SPA) | 설치 없는 배포, 파일 하나로 완결 |
| 스타일 | 순수 CSS3 (CSS 변수 기반 디자인 토큰) | Clinical Blue/Indigo 테마, 인쇄 전용 스타일(@media print) |
| 로직 | Vanilla JavaScript (ES2020+) | 프레임워크 없음 — 의존성·빌드 과정 제로 |
| AI 출제 | **Anthropic Claude API** (`claude-sonnet-5`, Messages API) | 학습내용 기반 문항 생성, JSON 구조화 출력 |
| 문서 텍스트 추출 | [JSZip](https://stuk.github.io/jszip/) | docx·pptx·hwpx (ZIP 기반 XML 문서) 본문 추출 |
| | [PDF.js](https://mozilla.github.io/pdf.js/) (Mozilla) | PDF 텍스트 추출 |
| | [SheetJS](https://sheetjs.com/) (xlsx) | 엑셀·CSV 시트 데이터 추출 |
| 저장 | Web Storage API (localStorage) | 문제은행, API 키 보관 |
| 내보내기 | Blob API + HTML 기반 .doc | Word 파일 다운로드 |
| 폰트 | Google Fonts — Gowun Batang(제목 명조) + IBM Plex Sans KR(본문) | 시험지 인쇄물 느낌의 타이포그래피 |

**설계 원칙**
- 추출 라이브러리는 해당 형식의 파일을 처음 첨부할 때만 CDN에서 **지연 로드**(lazy load) — 평상시 로딩 속도에 영향 없음
- 상태는 불변(immutable) 갱신 패턴으로 관리 — 셔플·편집 시 원본 훼손 방지
- AI 응답은 JSON 스키마 검증(보기 5개, 정답 번호 1~5 등)을 통과해야 렌더링 — 형식 오류 시 명확한 오류 메시지 표시
- 디자인 시스템은 자매 도구인 [PICO 기반 EBP 문헌검색 훈련기](https://jp5678.github.io/pico-search/)와 동일한 Clinical Blue/Indigo 팔레트 사용

## 5. 사용 흐름 요약

```
① 학년도·학기 → ② 학년·교과목 → ③ 시험 구분·시험 일시 → ④ 문항 수·배점 → ⑤ 난이도
→ ⑥ 학습내용 입력 + 수업자료 파일 첨부 → ⑦ 출제 방식 선택 → [시험문제 출제]
→ 시험지 확인 → 문항 클릭 수정 / 문항·보기 섞기 → 정답·해설 탭 검토
→ 인쇄(PDF) 또는 Word 저장 → 문제은행에 보관
```

## 6. 유의사항 및 제한

- **인터넷 연결**: API 출제, 파일 텍스트 추출(라이브러리 로드), 웹폰트에 필요. 이미 출제된 시험지의 열람·편집·인쇄는 오프라인에서도 가능
- **HWP(구형) 미지원**: 한글 문서는 hwpx 또는 PDF로 저장 후 첨부
- **스캔 이미지 PDF**: 텍스트가 없는 스캔본은 추출 불가(OCR 미지원) — 안내 메시지 표시
- **문제은행 저장 위치**: 브라우저별·컴퓨터별로 분리 저장되므로, 다른 컴퓨터로 옮길 때는 Word 저장본을 함께 보관 권장
- **AI 출제 특성**: 생성된 문항은 반드시 교수자가 검토 후 사용 (정답 오류·모호한 표현 가능성)
- **API 비용**: Claude API 모드는 Anthropic 계정의 사용량 요금이 발생 (시험 1회 출제당 소액)

## 7. 활용 시나리오

### 교수자 활용법

**① 정규 시험 출제의 초안 도구로 (핵심 용도)**
- 중간·기말고사 2~3주 전, 해당 학기 강의노트·PPT를 첨부해 문항 초안 출제 — 시험 일시를 캘린더로 지정하면 시험지 머리표까지 완성
- "문항·보기 섞기"로 A형/B형 두 벌을 만들어 부정행위 방지
- 핵심 흐름은 **AI 초안 → 교수 검토·수정 → 확정**. 화면에서 바로 고칠 수 있어 검토 시간이 짧으며, 정답 오류와 모호한 보기는 반드시 확인

**② 진단평가·형성평가 제작 (시험 구분에서 선택)**
- 학기 초 "진단평가"를 선택해 선수 지식 점검 문제 출제 — 참/거짓 유형이 빠른 진위 점검에 적합
- 매주 "형성평가"를 선택하고 해당 주 강의자료 하나만 첨부해 객관식·참/거짓 5문항 미니 퀴즈를 5분 안에 제작
- 난이도 "하 위주" = 수업 도입부 복습 퀴즈, "상 위주" = 토론용 심화 문제

**③ 문항 품질·난이도 보정**
- 같은 학습내용으로 난이도 분포를 바꿔 여러 번 출제한 뒤 좋은 문항만 골라 조합
- 문제은행에 학기별로 저장해두면 다음 학년도 출제 시 기출 중복을 비교 가능

**④ 수업 자체의 교보재로**
- 두 교과목 모두 AI 활용이 주제이므로 플랫폼 자체가 살아있는 수업 사례
- "프롬프트 복사 출제" 모드의 출제 프롬프트를 학생에게 보여주면 역할 부여·출력 형식 지정·범위 제한 등 프롬프트 설계 원리를 그대로 교육 가능

### 학습자 활용법

**① 시험 대비 자가 모의고사**
- 자기 필기·강의자료로 모의시험을 만들어 풀고 "정답·해설" 탭으로 즉시 채점
- API 키가 없어도 "프롬프트 복사" 모드로 무료 claude.ai에서 사용 가능

**② 스터디 그룹 상호 출제**
- 조별로 서로 다른 주차 범위를 맡아 문제를 만들고 교환해 풀기
- 문제를 만들어보는 것 자체가 상위 수준 학습(Bloom의 '창안')

**③ 서술형 답안 훈련**
- 서술형 문항의 채점기준(루브릭)이 배점과 함께 생성되므로, 자기 답안을 루브릭에 대조하며 채점자 관점을 학습 — 국가고시 서술 대비에도 유용

**④ AI 리터러시 실습 과제로**
- "AI가 만든 문항에서 오류·모호함을 찾아 수정하라"는 과제를 통해 AI 출력물을 비판적으로 검증하는 능력(두 교과목의 핵심 학습성과)을 직접 훈련

### 운영 시 권장사항

- 정규 시험은 반드시 **교수자 최종 검토** 후 사용하고, 확정본은 Word 저장으로 보관 (문제은행은 브라우저에만 저장됨)
- 학생 배포 시 API 키는 절대 공유하지 말고, 학생에게는 "프롬프트 복사" 모드를 안내
- 시험 보안이 중요한 문항은 학생 실습용 범위와 분리해서 관리
